【前提・ルール（必ず守る）】

このプロジェクトは fxbot（ミチビキ）です。

■ 役割分担
- ChatGPT：設計・仕様・影響範囲の決定
- Cursor：実装のみ（設計判断はしない）
- 人間：最終判断と確定

■ 参照すべきドキュメント（最優先）
1. docs/dev_rules.md
2. docs/public_api.md
3. docs/api_catalog.md

これらに反する実装は行わないでください。

■ アーキテクチャの絶対ルール
- GUI → services のみ呼び出し可
- services → core のみ呼び出し可
- GUI → core の直接アクセスは禁止
- core は services / GUI を import しない

■ API運用ルール
- 既存APIを最優先で使用する
- 新規関数・新規クラスは最小限
- 勝手にユーティリティ関数を増やさない
- public API は docs/public_api.md に定義されたもののみ
- core 層は内部実装。public にしてよいのは契約済みAPIのみ

■ 実装ポリシー
- 責務を跨ぐ処理をまとめない
- 「便利だから」「簡単だから」という理由で設計を変えない
- 仮実装・TODO を勝手に残さない
- 仕様にない挙動は追加しない

■ 出力ルール
- 変更したファイル名と変更点を明示する
- どの既存APIを使ったかを説明する
- 動作確認方法（実行コマンド）を必ず書く

以上を前提として、以下の指示を実装してください。
